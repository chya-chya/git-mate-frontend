# Git-Mate 프론트엔드 아키텍처 가이드

이 문서는 `git-mate-backend` 시스템을 시각화하고 사용자에게 분석 결과(PR, 통계, 역량 지표 등)를 제공하는 프론트엔드 애플리케이션의 구조와 기술적 결정 사항을 설명합니다.

## 1. 개요 (Overview)

Git-Mate 프론트엔드는 사용자의 GitHub 데이터를 기반으로 산출된 LLM 분석 결과를 인터랙티브하게 제공하는 것을 목표로 합니다. 데이터의 정밀함과 현대적인 사용자 경험을 조화롭게 구성하는 데 중점을 둡니다.

## 2. 기술 스택 (Tech Stack)

| 레이어          | 기술                    | 상세 이유                                                    |
| :-------------- | :---------------------- | :----------------------------------------------------------- |
| **코어**        | Next.js 15 (App Router) | SSR/ISR을 통한 성능 최적화 및 최신 라우팅 시스템 활용        |
| **상태 관리**   | Zustand                 | 간단하고 가벼운 클라이언트 상태 관리 (보일러플레이트 최소화) |
| **데이터 페칭** | TanStack Query v5       | 서버 상태(API 캐싱, 로딩/에러 핸들링)의 효율적 관리          |
| **스타일링**    | Tailwind CSS            | 유연한 디자인 시스템 구축 및 빠른 스타일링 적용              |
| **UI 컴포넌트** | Shadcn UI               | Radix UI 기반의 접근성 높고 커스터마이징이 용이한 UI         |
| **시각화**      | Recharts                | 가중 평균 및 누적 역량 지표의 직관적 차트 구현               |

## 3. 디렉토리 구조 (Directory Structure)

```text
src/
├── app/                 # Next.js App Router (페이지 및 레이아웃 관리)
├── components/          # 재사용 가능한 UI 컴포넌트
│   ├── ui/              # Shadcn UI (Atomic Components)
│   ├── layout/          # GNB, Sidebar, Footer 등 공통 레이아웃
│   ├── dashboard/       # 대시보드 위젯 및 차트 연동 컴포넌트
│   └── repository/      # 저장소 리스트 및 상세 분석 결과 뷰
├── hooks/               # API 통신 및 도메인 로직 커스텀 훅
├── services/            # Axios 기반의 백엔드 API 인터페이스 클래스
├── store/               # Zustand 전역 상태 (UserSession, UI 상태 등)
├── types/               # Backend DTO 및 Frontend 전용 인터페이스 정의
└── utils/               # 날짜 처리, 데이터 가공 및 포맷팅 유틸리티
```

## 4. 데이터 흐름 및 상태 관리 전략

- **Server State**: GitHub 레포지토리 목록, 분석 리포트 등의 데이터는 `TanStack Query`를 사용하여 서버에서 직접 관리합니다.
- **Client State**: 사용자의 테마 설정(Dark/Light), 현재 선택된 필터 정보, 모달 오픈 여부 등은 `Zustand`를 사용합니다.
- **Optimistic Updates**: 분석 시작 요청 시 권한 획득 후 즉시 UI에서 '분석 중' 상태로 표시되는 낙관적 업데이트를 적용하여 사용자 경험을 향상시킵니다.

## 5. 주요 인터페이스 설계 주의사항

- **반응형 디자인**: 개발자가 다양한 기기(데스크탑, 태블릿, 모바일)에서 자신의 분석 리포트를 확인할 수 있도록 완벽한 반응형 레이아웃을 지원합니다.
- **접근성(a11y)**: 차트 데이터에 대한 스크린 리더 지원 및 키보드 네비게이션을 기본 사양으로 포함합니다.
- **다크 모드**: IDE 환경과 유사한 편안한 분석 경험을 위해 다크 모드를 기본 옵션으로 제공합니다.
